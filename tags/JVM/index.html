JVM

04 Apr 2023 in Java

1、Java内存区域

主要有五大部分：程序计数器（私有）、虚拟机栈（私有）、本地方法栈（私有）、堆（公有）、方法区（公有），不过在8之后，方法区就已经被元空间代替了，而元空间在本地内存。

1、程序计数器

当前线程执行的字节码的行号指示器，什么循环、分支、线程恢复都靠这个。

2、虚拟机栈

每个方法调用的时候都会生成一个栈帧，栈帧里面存储了局部变量表，操作数栈、动态连接、方法出口等。局部变量表里面放了基本类型的引用，对象引用，

3、本地方法栈

和虚拟机栈很像，区别是为了虚拟机使用本地native方法服务的。

4、堆

虚拟机管理的内存中最大的一块，几乎所有的对象实例都在这里。这里有一块是字符串常量池，存的是堆String实例的引用。

5、方法区

存储被虚拟机加载的类型信息、常量、静态变量，编译后的代码缓存。不过在8之后就移到元空间了。

运行时常量池：

Class文件除了类的版本字段方法接口描述外，还有一个常量池，用于存放编译期间生成的字面量和符号引用，而当类加载到内存中后，jvm把符号引用换成直接引用，将class常量池中的内容存放到运行时常量池中。

对象的创建

1、当遇到new，先去检查这个指令的参数能不能再常量池找到类的符号引用，然后去检查是不是已经类加载、解析、初始化。

2、如果没有，就先执行类加载。

3、然后分配内存，内存的大小在类加载的时候就确认了。具体分配用“指针碰撞”还是“空闲列表”就看用那种垃圾收集器。为了保证不出现并发问题：有两种方案。1）CAS+失败重试。2）给线程自己预先分配本地线程分配缓冲。（TLAB）

对象的内存布局

每一个对象在堆里面分为三块：对象头、实例数据、对其填充。

对象头是markword+类型指针+数组长度（如果是数组的话），

  • markword就是记录哈希码、gc分代、锁标志、锁id，偏向锁。
  • 类型指针就是指向类型元数据的指针

对象的访问定位

1、句柄方法。reference放的是存储的是句柄地址，句柄包含对象实例数据和类型数据，各自的具体位置。句柄最大的优势就是即使在垃圾收集的时候对象被移动了，只用修改句柄中的指针就好，不用修改reference。

2、直接指针。存储的就是对象地址。他的优势是少一次间接访问的消耗。

2、内存溢出异常

除了程序计数器之外，其他几个运行时区域都有可能发生oom异常。

1、堆溢出oom

不断创建对象，然后避免垃圾回收

2、虚拟机栈和本地方法栈溢出sof

栈的深度超过最大深度，或者栈的容量无法申请到内存

3、方法区和运行时常量池溢出oom
